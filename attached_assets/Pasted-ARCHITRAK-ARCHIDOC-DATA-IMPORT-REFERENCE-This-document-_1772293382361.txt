ARCHITRAK — ARCHIDOC DATA IMPORT REFERENCE

This document describes every database field available in ARCHIDOC that ArchiTrak can import for financial tracking purposes. ARCHIDOC uses PostgreSQL with UUID-based primary keys (varchar(255)) and timestamps. All monetary values in quotation tables are stored as integers (cents). Proposal fee values use numeric(12,2) for precision. The database is accessed via Drizzle ORM.

1. PROJECTS

KEY NOTES FOR ARCHITRAK INTEGRATION
Monetary values: Quotation tables store amounts in integer cents (divide by 100 for display). Proposal fee tables use numeric(12,2) — already in euros with 2 decimal places.

Lot codes: The French construction lot system (LOT1 = Demolition, LOT2 = Gros Oeuvre, etc.) is used throughout to categorise work, quotations, and tasks. ArchiTrak should import and respect these codes for cost categorisation.

Primary keys: Projects, contractors, tasks, and DQE items use UUID strings (varchar(255)). Quotations, line items, and F&F items use auto-incrementing serial integers.

No existing invoice/payment tables: ARCHIDOC has no tables for invoices, payment schedules, or payment tracking. This is the gap ArchiTrak will fill.

Quotation lifecycle: awaiting -> received -> accepted/rejected/revised. ArchiTrak can use the accepted status as the trigger for creating payment schedules.

Shared database option: If ArchiTrak shares the same PostgreSQL database, it can reference ARCHIDOC tables directly via foreign keys. The database is Neon-backed PostgreSQL.
