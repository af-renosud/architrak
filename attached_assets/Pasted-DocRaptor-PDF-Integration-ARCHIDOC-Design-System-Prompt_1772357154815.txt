DocRaptor PDF Integration — ARCHIDOC Design System Prompt

This application is a sister product to ARCHIDOC (an architectural project management OS). All PDF reports must share the same professional "High-End Architectural" aesthetic. Use DocRaptor with the PrinceXML engine for PDF generation. Below is the complete design specification.

DocRaptor Integration

Send HTML+CSS to DocRaptor's API (https://docraptor.com/docs) to produce PDF output
Use prince_backend as the document engine — this enables advanced @page rules, running headers/footers, and proper page numbering via CSS counters
All styling is inline CSS embedded in the HTML — no external stylesheets

Typography

Body text: 10pt, color #34312D, line-height: 1.55
Main titles: 18pt, weight 700, color #0B2545
Section headers: 12pt, weight 700, uppercase, letter-spacing: 0.08em, color #0B2545
Metadata/footer text: 7pt–9pt, color #7E7F83
If the Inter font is unavailable, fall back to a stylized "ARCHIDOC" text header in Navy with letter-spacing: 1.5pt
Page Layout (@page rules)

@page {
  size: A4;
  margin: 22mm 18mm 25mm 18mm;
}
Running header: Logo top-left, document title top-right (uppercase, grey, small)
Running footer: Project/client info left, date center, "Page X / Y" right
Page numbering via PrinceXML CSS counters: content: "Page " counter(page) " / " counter(pages)

General Principles

Clean, airy layouts with generous whitespace — aim for an architectural drawing aesthetic
Section separation using margin-bottom: 6mm minimum between blocks
All financial figures should use monospace or tabular number formatting
Currency formatting: Euro (€) with French locale (e.g., 1 234,56 €)
Date formatting: French locale (dd/mm/yyyy)
Maintain high contrast for readability — never use light text on light backgrounds
PDF file names should follow the pattern: [TYPE]-[PROJECT]-[DATE].pdf
